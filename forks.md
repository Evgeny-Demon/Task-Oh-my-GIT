# __```О форках```__
Fork в **__`Git`__** (чаще всего — на __`GitHub`__, __`GitLab`__ и подобных платформах) — это *`создание личной копии`* чужого репозитория. Эта копия появляется в вашем аккаунте и полностью независима от оригинала, но сохраняет связь с ним, чтобы можно было предлагать изменения обратно.

#### *`Зачем нужен fork?`* 
Чтобы безопасно экспериментировать с чужим кодом, не имея прав на запись в оригинальный репозиторий.
Чтобы внести улучшение и предложить его автору через Pull Request (PR).

#### *`Как это работает:`*
    1. Леонардо опубликовал репозиторий tmnt-battle-system на GitHub.
    2. Донателло хочет добавить поддержку дронов, но не может писать напрямую в репозиторий Леонардо.
    3. Донателло нажимает Fork — у него появляется копия репозитория: Donatello/tmnt-battle-system.
    4. Он клонирует свою копию, создаёт ветку, делает изменения, пушит их к себе.
    5. Затем он создаёт Pull Request из своей ветки в оригинальный репозиторий Леонардо.
    6. Леонардо просматривает изменения и, если всё в порядке, принимает их.

 ## *Важно:*
Fork — это одноразовое копирование.
Fork живёт на сервере, а не только локально.
Fork ≠ clone: [*`clone`*](/git_clone.md) — это копия на вашем компьютере, fork — копия на сервере в вашем аккаунте.
