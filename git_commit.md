# __```git commit```__
---
Поговорим о снаряжении. Представь, что ты упаковал нужное тебе снаряжение с помощью команды [git add](./add.md), и теперь нужно, чтобы Леонардо зафиксировал сие действо: 

```bash=
git commit -m "Короткое описание изменений"
```
И можно отправляться в путь.

Команда `git commit` создаёт снимок __*(snapshot)*__ состояния всех файлов, которые ты добавил в индекс, и сохраняет его в историю репозитория. Это как оставить отметку в дневнике миссий: «Сегодня мы починили ловушку для Шреддера и добавили перец в рецепт пиццы».

```bash=
git commit -m "Добавил ингредиенты для огненной пиццы"
```
Без __**-m**__ Git откроет текстовый редактор, чтобы ты написал сообщение вручную (но для начала удобнее использовать __**-m**__).

`git commit` сохраняет только то, что было добавлено через [git add](./add.md)  

Каждый коммит имеет уникальный хеш (например, **a1b2c3d**) — как отпечаток миссии.

__***`ВАЖНО!!!`***__ Сообщение коммита должно быть понятным, чтобы и тебе через неделю, и Сплинтеру с Леонардо завтра было ясно, что изменилось.
