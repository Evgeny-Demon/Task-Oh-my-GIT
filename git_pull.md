# __```git pull```__

Командой `git pull` разработчик получает последние изменения из удалённого репозитория и сливает их в свою текущую ветку — как будто Леонардо обновляет код базы, а остальные черепашки подтягивают его правки к себе.

Допустим, Микелянджело работает над веткой __*`pizza-feature`*__ и хочет получить свежие изменения от Донателло. Он пишет:
```bash=
git pull
```
Или Раф хочет явно подтянуть последнюю версию основной ветки от Леонардо. Тогда Раф пишет:
```bash=
git pull origin main
```
Команда особенно полезна, когда черепашки совместно правят один проект и хотят держать свои локальные копии в актуальном состоянии.
